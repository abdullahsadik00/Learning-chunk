# Progressive Assessment System — Days 1–35

## How to use

1. Complete the teaching file for a day.
2. Open the matching assessment file (`day-01.md`, `day-02.md`, …).
3. Write your solution for each question **before** looking anything up.
4. Check your solution against the **Acceptance Criteria** checklist.
5. Count how many criteria you fully satisfied — that is your score.

---

## Scoring rubric (per file, 15 questions max)

| Score | Stage | What it means |
|-------|-------|---------------|
| 0–4   | 🔴 Re-study | Go back and re-read the teaching file. Take notes. |
| 5–9   | 🟡 Getting there | You understand the basics but concepts aren't solid yet. Redo the examples. |
| 10–12 | 🟢 Solid | You have a working understanding. Move on, revisit gaps later. |
| 13–15 | 🚀 Ready to advance | Strong grasp. You can confidently use this in real work. |

## Tips

- Treat every question as a real task, not an exam. Write actual code in a scratch file.
- If you can't write it, can you explain it out loud? If not → re-study.
- A Solid score today + re-visit in 2 weeks = long-term retention.
- The difficulty markers (⭐ ⭐⭐ ⭐⭐⭐) reflect real interview signal:
  - ⭐ Expected in any junior role
  - ⭐⭐ Expected in mid-level interviews
  - ⭐⭐⭐ Expected in senior / tech-lead rounds
